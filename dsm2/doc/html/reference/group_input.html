GROUP Section

GROUP is a section where the user declares group names that will be used for the outputs. Group members are defined in a separate GROUP_MEMBER sections. A group is a collection of model objects, for instance groups of water bodies or groups of boundary inputs. Groups are used a number of places, including: tracking of constituents originated from grouped sources, tracking of particles as they reside or move between groups, and assignment of non-conservative constituents parameters to groups.

Reference

Keyword Descriptions

NAME
Name of the group. No space is allowed.

Examples:

GROUP
NAME
twitchell
emmaton
ag_div# Comment: Agricultural Diversion
swp
mtz
END

Usage Notes:

  • The maximum number of groups is limited in the DSM2 code.
