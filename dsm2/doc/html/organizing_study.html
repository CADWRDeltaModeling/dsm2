Organing a Study

Overview:

Within the database, DSM2 projects are grouped into simulations and models. The simulation is the top level organizational structure in DSM2: when you open up the user interface (GUI), the first thing you will do is open, clone or create a simulation.

Organizationally, a simulation is a package of related model runs with compatible geometry and flow input. A simulation always includes a DSM2-HYDRO model run to establish a flow field. Optionally, it may also contain one or more water quality and particle tracking models. The models are not tightly coupled -- flow data is transfered from HYDRO to QUAL or PTM through a file called a tidefile.

A model is one computer model component of a simulation, the input for one DSM2-HYDRO, QUAL or PTM run. If there is a simulation described as planning there might models called planning_hydro, planning_qual_ec and planning_qual_do that describe hydrodynamic and water quality models that would be launched independently.

Simulations and models are containers. The actual data such as channels, gates or boundary conditions are grouped in sets called layers. Layers are a means of grouping like components. The layering system is described in more detail in the Layers documentation
