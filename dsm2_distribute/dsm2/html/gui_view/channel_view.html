Channels

Overview:

Channels are the fundamental objects of the DSM2 grid. The Channels View of the GUI allows you to enter channel connectivity, parameters and geometry. Channels connectivity is defined by an upstream and downstream node numbers of the channels. Two child tables in the Channels View describe the locations and geometry of user-described cross-sections in the selected channel. Note that a default initial condition is required for every channel number in the DSM2 grid, and this is entered seperately in the Channel Initial Conditions View.

Channel Table

Channel table defines the connectivity, length, friction and dispersion characteristics of a channel. The channel table is a top-level layered table.

Field Descriptions

Layer
Layer number on which the channel entry is being made.
Channel Number
Channel number. This is the identifier*** of the channel, and corresponds to the number you typically see on a grid map.
Length (ft)
Length of the channel reach
Mannings n
Mannings friction coefficient for the whole reach.
Dispersion
Dimensional dispersion factor.
Node Up
Number of the upstream node at which channel is connected.
Node Down
Number of the downstream node at which channel is connected.
Use
Check box indicating item is in use. See Use Column.

Channel Cross Section Table

This table lists locations where bathymetric cross-sections are specified by the user. The table lists the fraction of the distance along the reach (from upstream to downstream) at which the user cross-section is located. These cross-sections will be interpolated by the model at computational points. Overspecification of geometry is a frequent source of user error/misconception, please see usage notes below

Field Descriptions

Distance (fraction)
Fraction of distance from upstream node to downstream node where cross-section is located
Copied From [optional]
Column to describe channel and distance from which cross section was copied or derived, in case the cross-section is not original.
Description[optional]
Optional user data describing the origin of the cross-section.

Cross Section Layer Table

The Cross-Section Layer Table lists geometric information about each cross-section. This information is in the form of lookup tables of hydraulically important quantities such as area, width and wetted perimeter.

Field Descriptions

Elevation
Elevation from bottom at which properties are known. The area, width, etc. apply to this elevation, and channel properties between elevations are linearly interpolated.
Width
Width of channel at top (ft).
Area
Area of channel from bottom to cross section(sq ft). Ignored if Area disagrees at high precision with the integral of Width.
Wetted Perimeter
Wetted perimeter of channel at given elevation.

Usage Notes:

  • All channels must have an initial condition and at least one cross-section.
  • Older versions of DSM2 had the notion of a "regular" cross-section (meaning rectangular). In the current DSM2 this is just a cross-section with two layers.
  • Users frequently overspecificy cross-sections, either by specifying more cross-sections longitudinally than the model can possibly use or by describing cross-sections vertically in such a way as to capture highly local features such as small constrictions, sills and undulations. DSM2 is commonly used with spatial resolution (delta x) of several thousand feet. You should only include features that are well resolved by this resolution, which means changes that persist over several miles. Even more importantly, you should avoid adjacent cross-sections with bottoms that vary greatly in elevation because they can cause odd behavior when cross-sections are interpolated to computation points. The bottoms layers of cross sections should represent the "overall" slope of the channel.
  • Tools are available for importing cross-sections from CSDP. However, you should be aware of the accuracy problem in CSDP -- the cross-section area reported by this program is not accurate, and is therefore ignored by DSM2 which recalculates area by integrating width.
