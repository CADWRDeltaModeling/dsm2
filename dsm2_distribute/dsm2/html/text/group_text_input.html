GROUPS Section

GROUPS is a section that allows the user to describe groups of model objects, for instance groups of water bodies or groups of boundary inputs. Groups are used a number of places, including: tracking of constituents originated from grouped sources, tracking of particles as they reside or move between groups, and assignment of non-conservative constituents parameters to groups. The GROUPS Section is equivalent to the Groups View in the GUI.

Reference

Keyword Descriptions

GROUP_NAME
Name of the group whose members are augmented by the text line.  If the name does not match an existing group, a new group is created.
MEMBER_TYPE
Type of model object included in the group. 
MEMBER_ID
A regular expression that matches the identifier of the members.

Examples:

GROUPS
GROUP_NAMEMEMBER_TYPEMEMBER_ID
twitchellchannel4[5-8]# matches channels 45-48
twitchellchannel(83|280|310)# matches three channels
emmatonchannelrange:430-434# no spaces allowed
ag_divqextdicu_div_.*# dot-star is a wildcard
ag_divqextbbid.*
swpqextswp
mtzstagemtz.*# matches stage input name
END

Usage Notes:

  • It is not recommended that you mix group member assignments between text and GUI.
