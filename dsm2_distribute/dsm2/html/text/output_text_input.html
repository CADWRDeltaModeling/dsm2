OUTPUTPATHS Section

Overview:

Output paths are used to request time-varying output in HEC-DSS or text format. Such output is equivalent to requests made in the output section of the GUI (views such as Channel Output and Gate Output).

Text OUTPUTPATHS are the preferred place to make study or scenario-specific output requests -- layers should be created in the GUI only to handle regular, cohesive sets of output that will be used frequently or in multiple contexts.

Reference

Keyword Descriptions

NAME
Name of the output. This is the identifier attached to the output. It will be used as the B_PART of DSS output. Note: the meaning of this field has changed significantly from previous versions of DSM2 -- it has nothing to do with the location being output.
CHAN
Channel number where output is being requested. Applies only to output from channels, and cannot be mixed in the same input block with gate or reservoir identifiers.
[DIST]
Distance along channel at which output is requested.
GATE
Gate at which output is requested.
[DEVICE]
Gate device at which output is requested. May be "none" if mixing whole-gate requests with device specific requests.
RESERVOIR
Reservoir at which output is requested.
[RESERVOIR_NODE]
Reservoir connected node at which output is requested. May be "none" if mixing whole reseroir requests with reservoir-node requests
TYPE
Variable being output.
INTERVAL
Frequency at which output is requested, in the form of a DSS-compatible interval that is not calendar dependent. The units can be MIN, HOUR, DAY, e.g.: 15MIN, 1HOUR, 1DAY
PERIOD
Aggregation that is applied to convert variables from the model time step to the output interval.: use INST for an instantaneous snapshot and AVE to get a period average.
FILENAME
Name of file in which to store output. The file will be created if it does not exist already.

Examples:

The following example selects channel data and uses an ENVVAR reference for the file name.

OUTPUTPATHS
NAMECHANDISTTYPEINTERVALPERIODFILENAME
MTZ441LENTHSTAGE15MININST${OUTDSS}
RIO_VISTA4309684FLOW1HOURINST${OUTDSS}
END

The following example selects gate data.

OUTPUTPATHS
NAMEGATEDEVICETYPEINTERVALPERIODFILENAME
MIDR_BARRIERMIDDLE_R_BARRIERNONEINSTALL15MININST${OUTDSS}
MIDR_BARRIERMIDDLE_R_BARRIERBARRIERWIDTH1HOURINST${OUTDSS}
MIDR_BARRIERMIDDLE_R_BARRIERBARRIERELEV1HOURINST${OUTDSS}
MIDR_BARRIERMIDDLE_R_BARRIERBARRIERHEIGHT1HOURINST${OUTDSS}
MIDR_BARRIERMIDDLE_R_BARRIERBARRIERFLOW1HOURINST${OUTDSS}# careful of orientation
END
