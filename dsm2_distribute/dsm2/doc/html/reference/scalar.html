SCALARS Section

Overview:

SCALARS are text input variables used to specify model-wide numerical properties and echoed output levels. Scalar text input is equivalent to variables specified in the Parameter View of the GUI. Text is the preferred way to make changes to scalars on a single-project or single-scenario basis.

All production DSM2 model runs have a scalar section. Several fields are particularly important even if most of the input comes from the database. The database parameter (default "dsm2input") names the database. The model_name parameter specifies the model name in the database from which to load other input. The start and end times are also specified in text (usually with an ENVVAR text substitution) and have no default in the database.

Reference

The Parameter table comprises name-value pairs for scalars. The scalars that are allowed depend on the specific model. SCALAR entries in text replace the equivalent GUI entries.

Keyword Descriptions

NAME
Name of the parameter. This is the identifier of the parameter.
VALUE
Value assigned to the parameter. These are interpreted by the model first as text (to allow substitution using ENVVARS) and then converted to the correct data type and validated.

Examples:

HYDRO example:

SCALAR
NAMEVALUE
model_namehistorical_hydro# model name in the database
run_start_date${START_DATE}# runtime using envvars
run_end_date${END_DATE}
run_start_time${START_TIME}
run_end_time${END_TIME}
END

QUAL example:

SCALAR
NAMEVALUE
databasedsm2input_access#dbase name (ODBC name)
model_namesdip_qual_ic#model name in the database
run_start_date${QUAL_START_DATE}
run_start_time${START_TIME}
run_end_date${END_DATE}
run_end_time${END_TIME}
temp_dirz:/temp# reassign temp directory
init_conc20.0# default initial concentration
END

Usage Notes:

  • The GUI Parameter View give a good quick reference of default values for a lot of scalar parameters.
