SCALARS Section

Overview:

SCALARS are text input variables used to specify model-wide numerical properties, runtime parameters and echoed output levels.

All production DSM2 model runs have a scalar section. The start and end times are specified here (often with an ENVVAR text substitution) and these no default.

Reference

The SCALAR table comprises name-value pairs for scalars. The scalars that are allowed depend on the specific model.

Keyword Descriptions

NAME
Name of the parameter. This is the identifier of the parameter.
VALUE
Value assigned to the parameter. These are interpreted by the model first as text (to allow substitution using ENVVARS) and then converted to the correct data type and validated.

Examples:

HYDRO example:

SCALAR
NAMEVALUE
run_start_date${START_DATE}# runtime using envvar
run_end_date${END_DATE}
run_start_time${START_TIME}
run_end_time${END_TIME}
END

QUAL example:

SCALAR
NAMEVALUE
run_start_date${QUAL_START_DATE}
run_start_time${START_TIME}
run_end_date${END_DATE}
run_end_time${END_TIME}
temp_dirz:/temp# reassign scratch dir
init_conc20.0# default global concentration
END

Usage Notes:

  • Most users will want to use dss_direct=true to avoid writes to the scratch directory
